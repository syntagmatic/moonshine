The Scorecard

How autoresearch audits D3 blocks across four quality dimensions

Every block in d3-power-tools goes through an automated audit before it gets a score. The auditor is a set of four independent Claude subprocesses, each evaluating a different aspect of visualization quality. The result is a 4-dimension scorecard that gets compressed into a single composite number.

A single composite score hides real tradeoffs. A block can score 7.0 while having a broken layout, because strong scores on encoding integrity and stress test mask a weak visual critic score. Understanding the dimensions separately reveals what the number actually means.

The four dimensions

Visual Critic

weight: 0.30

Design quality. Color harmony, typographic hierarchy, whitespace, data-ink ratio, overall feel. A 5 or 6 means functional but generic. A 7 or 8 means deliberately designed.

Encoding Integrity

weight: 0.25

Data honesty. Lie factor, zero baselines, scale appropriateness, data join correctness. This is about whether the chart tells the truth.

Cognitive Load

weight: 0.25

Clarity. Working memory limits, animation congruence, visual complexity, label legibility. Can a reader understand this in a few seconds?

Stress Test

weight: 0.20

Interaction robustness. Update storms, stale closures, feedback loops, transition conflicts. Does it break when you click fast?

The composite formula is:

composite = 0.30 * vc + 0.25 * ei + 0.25 * cl + 0.20 * st

Those default weights reflect a judgment call. Visual design carries slightly more weight because users see it first. Stress testing carries a little less because many blocks are demonstrations, not production tools. But these weights are adjustable. How the audit runs

Each dimension runs as a separate claude -p subprocess. The auditor receives two inputs: the block's HTML source and a screenshot of the rendered output. It reads both, scores the block 1 to 10 on its assigned dimension, and writes a short note explaining why it chose that score.

The four subprocesses run in parallel. When all four return, their scores are combined into the composite. If the composite falls below a threshold, the experiment is discarded. But the auditor notes do not disappear. They feed back to the proposer as context for the next attempt. "Visual critic: 4. The legend overlaps the y-axis labels and the color scale is not perceptually uniform." That kind of feedback gives the proposer something concrete to fix.

This feedback loop is the reason autoresearch converges. Without per-dimension notes, a low composite score is just a number. The proposer would have to guess what went wrong. With the notes, it knows exactly which dimension pulled the score down and why.

The scorecard is not a final judgment. It is a communication tool between two subprocesses. The auditor explains what is wrong. The proposer reads that explanation and tries again. The composite is just the summary statistic that decides whether to keep iterating.

How the scale was calibrated

The 1-to-10 integer scale did not arrive fully formed. Early versions of the auditor used a 1-to-5 scale, which collapsed into three effective grades: bad, fine, good. Everything clustered at 3. Expanding to 10 gave enough room for the auditor to separate "functional but generic" from "deliberately designed" without resorting to decimals.

Calibration happened through split audits. The first split (blocks 1 through 42) revealed that the visual critic was scoring too generously. A block with default axis labels and no color thought scored a 7, which is supposed to mean "deliberately designed." The fix was a scoring anchor: 5 means the chart works but nothing about its appearance was intentional. A 6 means one deliberate choice (a curated palette, or considered whitespace). A 7 or 8 means multiple deliberate choices that work together. 9 and 10 are reserved for blocks where every visual element earns its place.

Who decided these anchors? No external panel. The calibration was done by running the auditor across blocks with known quality levels and checking whether the scores matched human judgment. When they did not, the skill prompt was revised. The visual critic went through three prompt revisions before its scores stopped clustering at the high end. Encoding integrity required two revisions, mainly to stop penalizing tech demos for missing axis labels (a code quality issue, not a correctness bug).

The second split (blocks 43 through 84) surfaced a different problem: overlap between dimensions. The original set included six auditing skills, not four. A "robustness contract" skill overlapped heavily with stress test. A "deception detector" and a "metamorphic tester" were doing versions of the same work, checking whether the chart tells the truth under perturbation. The fix was to merge deception detector and metamorphic tester into encoding integrity, and to drop robustness contract entirely. That left four dimensions with minimal overlap.

Why four, specifically

Three dimensions would require each one to cover too much ground. Visual critic would need to evaluate both aesthetics and cognitive clarity in the same pass, which are genuinely different concerns. A chart can be beautiful and confusing. Eight dimensions would give the proposer too many knobs to optimize against, and empirically the auditor's scores became less consistent when asked to evaluate narrow aspects like "transition smoothness" in isolation.

Four is the pragmatic middle. Each dimension maps to a recognizable category of visualization quality. Each is broad enough to score reliably but narrow enough that a low score gives the proposer an actionable direction. "Visual critic: 4" says to fix the look. "Encoding integrity: 4" says the chart is lying. Those are different repairs.

The subprocess architecture

Each dimension runs as a separate claude -p invocation. The alternative would be a single prompt that evaluates all four at once, but single-pass evaluation showed score leakage: a striking visual design inflated the encoding integrity score. Separation forces each subprocess to focus. The visual critic never sees the stress test prompt. The encoding integrity auditor never sees the visual critic's score.

The four subprocesses receive identical inputs: the block's HTML source code and a screenshot of the rendered output. Each subprocess has its own system prompt, tuned during calibration, that defines what it should look for and the scoring anchors for its dimension. Each returns a JSON object with an integer score (1 to 10) and a note (free-text, typically 2 to 4 sentences). The four calls run in parallel. Wall-clock time for an audit is the time of the slowest subprocess, usually 8 to 15 seconds.

History of the composite weights

The first composite was an unweighted average: 0.25 across all four dimensions. This was intentionally naive. Equal weights make no assumptions about which dimension matters more.

The shift to 0.30 / 0.25 / 0.25 / 0.20 came after analyzing the first 40 audited blocks. Two patterns emerged. First, blocks with strong visual critic scores were consistently rated higher by human reviewers, even when other dimensions were mediocre. Readers see the surface first. Second, stress test scores had the highest variance and the weakest correlation with human quality judgments, partly because many blocks are demonstrations, not production tools. A demo that breaks under rapid clicking is less of a problem than a demo that lies about data.

Giving visual critic a 0.30 weight and stress test a 0.20 weight aligned the composite more closely with human rankings across those 40 blocks. The change was small, 5 percentage points per dimension, but it shifted about 15% of pairwise rankings. The current weights have been stable since that adjustment.

How audit notes feed back to the proposer

The composite decides whether an experiment is kept or discarded. But the audit notes do the real work. When the proposer receives context for its next attempt, it gets the notes from every failed experiment since the last keep. A typical context block looks like: 

Experiment 7 (discarded, composite 6.3):
  visual_critic: 4 - Legend overlaps y-axis labels. Color scale
    is not perceptually uniform. Title is clipped on mobile.
  encoding_integrity: 7 - Scales correct, joins robust.
  cognitive_load: 7 - Clear layout, readable labels.
  stress_test: 6 - Tooltip flickers on rapid mousemove.

The proposer reads this and knows: fix the legend position, swap to a perceptual color scale, check the title overflow. It does not need to guess. This specificity is why the feedback loop converges. A bare composite of 6.3 would give the proposer nothing to work with. The per- dimension notes turn a number into a repair list.

The notes also accumulate. If three consecutive experiments fail on the same issue ("legend overlaps y-axis labels"), the proposer can see that its previous fix attempts did not work and needs to try a structurally different approach. Without the notes, it might keep generating the same broken fix.

Dimension correlations

If all four dimensions measured the same underlying quality, their scores would be perfectly correlated and there would be no reason to have four of them. The scatter matrix below shows pairwise relationships. Each cell plots one dimension against another, with one dot per block.

Two patterns stand out. Visual critic and cognitive load show moderate positive correlation (around 0.45): blocks with good design tend to also be clear. But visual critic and stress test are nearly uncorrelated (around 0.12). A block can look polished and still break under interaction pressure. Encoding integrity and stress test show weak correlation (around 0.25), which makes sense because data honesty and interaction robustness are genuinely different concerns.

The low correlations between dimensions confirm that four dimensions capture different information. Dropping any one would lose signal that the others do not recover.

What the scorecard hides

The composite compresses tradeoffs into a single number. A block scoring 8-4-8-8 across the four dimensions gets a composite of 7.0. So does a block scoring 7-7-7-7. The first block lies about its data (encoding integrity of 4). The second block is uniformly decent. The composite treats them as equal.

This is the fundamental limitation of any weighted average. The composite was designed to make keep/discard decisions during the loop, where a single threshold is operationally necessary. But when comparing blocks outside the loop, or when deciding which block to ship, the per-dimension breakdown matters more than the composite.

Weight sensitivity

The weights 0.30 / 0.25 / 0.25 / 0.20 were chosen to match human rankings across 40 reference blocks. But small changes to the weights can flip pairwise rankings. Consider two blocks: Block A scores (8, 6, 7, 7) and Block B scores (6, 8, 7, 7). At the default weights, Block A leads by 0.10 because its visual critic advantage (weight 0.30) outweighs Block B's encoding integrity advantage (weight 0.25). Equalizing the weights to 0.25 each flips the ranking because the dimensions tie.

In practice, about 12% of adjacent pairs in the ranking swap positions when the visual critic weight moves from 0.30 to 0.25, all else held constant. The interactive slider above lets you see this for the sample blocks. The full 107-block ranking is more sensitive because more blocks cluster near the same composite.

False precision of integer scales

The 1-to-10 scale looks precise, but the auditor's effective resolution is coarser. A score of 6 versus 7 on visual critic depends on whether the auditor considers a single curated palette to be "one deliberate choice" or "coordinated choices." Different runs on the same block can produce scores that differ by 1 point, sometimes 2. This means a composite difference of 0.2 or less is noise, not signal.

The ensemble auditing approach (covered in a later chapter) addresses this by running each dimension multiple times and averaging. That reduces variance but does not eliminate it. Any scoring system where the grader is a language model will have irreducible noise from prompt sensitivity and sampling temperature.

The floor of encoding integrity

An encoding integrity score below 5 should be treated differently from low scores on other dimensions. Visual critic at 4 means the block looks bad. Encoding integrity at 4 means the block actively misleads the reader about data. In the current system, both contribute to the composite through the same weighted sum. An alternative would be to treat encoding integrity below 5 as a hard veto, blocking the keep decision regardless of the composite. The current system does not implement this, and there are blocks in the corpus that compensate for poor encoding integrity with strong scores elsewhere. The composite says they are fine. They are not.
