The Attention Mechanism

How transformers decide which words matter for understanding each other word.

How does a model know which words matter for translating this one? In the sentence "The cat sat on the mat," understanding "sat" requires knowing who sat (the cat) and where (the mat). The words carry unequal importance, and that importance shifts depending on which word you are trying to understand.

Before attention, sequence models processed words in order, compressing everything into a single fixed-size vector. Long sentences lost information. Distant dependencies faded. Attention solved this by letting the model look at every word simultaneously and choose what to focus on, weighted by relevance.

This is not neural magic. At its core, attention is a weighted average. The trick is how the model learns which weights to assign.

From Hard Lookup to Soft Attention

Think about looking up a word in a dictionary. You have a query (the word you want), a set of keys (the dictionary entries), and the values (the definitions). In a hard lookup, your query either matches a key exactly or it does not. One hit, all or nothing.

Attention is a soft lookup. Instead of demanding an exact match, the query partially matches many keys at once. Each match gets a weight, and the output is a blend of all the values, mixed according to those weights. The better a key matches the query, the more its value contributes.

Attention replaces the binary yes/no of a dictionary lookup with a spectrum of relevance. Every key contributes, but relevant keys contribute more.

Queries, Keys, and Values

Each word in the input is represented as a vector, a list of numbers that captures its meaning. To compute attention, the model transforms each word into three different roles simultaneously.

The query represents what a word is looking for. The key represents what a word offers to be found by. The value represents the information a word contributes when it is attended to. These are produced by multiplying the input embedding by three learned weight matrices: WQ, WK, and WV.

The same word plays all three roles. "Cat" has a query (what is cat looking for?), a key (what makes cat findable?), and a value (what information does cat carry?). The model learns to separate these roles during training.

The weight matrices WQ, WK, and WV are the learnable parameters. The model starts with random projections and adjusts them during training so that useful relationships produce high attention scores.

Scaling and Softmax

Once we have queries and keys for every word, computing attention scores is straightforward: take the dot product of each query with each key. A high dot product means that query and key point in a similar direction. The result is a matrix of raw scores, one for each pair of words.

But raw dot products can grow large, especially in high-dimensional spaces. Large inputs to softmax produce extremely peaked distributions where one word gets nearly all the weight. Dividing by the square root of the key dimension (√dk) keeps the values in a range where softmax produces useful, spread-out distributions.

After scaling, softmax normalizes each row so the weights sum to 1. This turns raw affinity scores into a proper probability distribution. Finally, these weights multiply the value vectors to produce the output: a weighted blend of information from across the sentence.

Figure 3. The attention computation in four stages. Step through with the buttons or scroll. Click a row to inspect which words that query attends to. "Sat" attends heavily to "cat" (who sat?) and "mat" (sat where?).

Step 1: Raw Scores

The dot product Q × KT produces raw affinity scores between every pair of words. Larger numbers mean the query and key vectors point in more similar directions.

Step 2: Scale

Divide by √dk to prevent the dot products from growing too large. Without scaling, softmax would collapse into near-one-hot distributions.

Step 3: Softmax

Softmax normalizes each row to sum to 1, turning scores into attention weights. Each row is now a probability distribution over the sentence.

Step 4: Apply to Values

Multiply the attention weights by V to produce the output. Click any row to see which words that query attends to most strongly.

The attention heatmap is not something the programmer designs. It emerges from training. The model discovers on its own that "sat" should attend to "cat" and "mat" because doing so helps predict the next word or produce a correct translation.

Many Heads, Many Relationships

A single attention pattern can only capture one kind of relationship at a time. But language is full of overlapping structure: syntax, coreference, word order, semantic similarity. Multi-head attention runs several attention computations in parallel, each with its own learned Q, K, V projections. Each head is free to specialize.

One head might learn to track which noun a verb relates to. Another might focus on adjacent words for local syntax. A third might connect repeated words. The outputs of all heads are concatenated and projected back to the model's dimension.

This is why transformers are powerful. They do not commit to a single way of reading a sentence. They read it many ways simultaneously, and the model learns to combine those readings into a rich representation.

A Model's Reading Strategy

Attention is how a transformer reads. Not word by word from left to right, but all at once, with every word evaluating every other word for relevance. The mechanism is simple: project into queries, keys, and values; compute similarity; normalize; blend. The complexity comes from stacking this operation in layers and running it in parallel heads, each learning to notice different aspects of language.

What makes this work is that the model does not need to be told what to attend to. Given enough examples and a clear objective, it discovers on its own that verbs should look at their subjects, that pronouns should find their referents, that modifiers should bind to the words they modify. Attention is not a rule system. It is a learned reading strategy that emerges from the structure of language itself.
