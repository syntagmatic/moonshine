Backpropagation

A neural network learns by following blame backward through its connections.

Every neural network starts ignorant. Its weights are random numbers, its predictions nonsense. Learning means adjusting those weights so the network's output gets closer to the right answer. But a network might have thousands of connections. Which ones should change, and by how much?

Backpropagation answers that question. It takes the error at the output and traces responsibility backward through every connection, assigning each weight a share of the blame. Then each weight nudges itself in the direction that reduces the error. Repeat this thousands of times and the network learns.

The algorithm is mechanical. There is nothing mystical about it. But it helps to see it working.

The Forward Pass, Step by Step

Scroll through the steps below to see exactly what happens inside each node. The weighted sum is built piece by piece, then the sigmoid function squashes it into a bounded activation. At the end, the network's prediction meets the target and we measure how wrong it is.

Step 1

Input values enter the network. These are raw numbers provided from outside, not computed by the network itself.

Step 2

Each hidden node computes a weighted sum. Every incoming connection multiplies the input by its weight. The stacking bars show each contribution. Tall blue bars pull the sum up; red bars pull it down.

Step 3

The sigmoid activation function squashes each weighted sum into the range (0, 1). The curve shows where the sum lands and what activation comes out. Values far from zero get pushed toward 0 or 1; values near zero land around 0.5.

Step 4

The output node repeats the same process: weighted sum of hidden activations, then sigmoid. This gives the network's prediction.

Step 5

Loss measures the gap between prediction and target. The two bars sit side by side, and the highlighted gap between them is the error the network needs to reduce. Squared error penalizes large mistakes more than small ones.

Measuring How Wrong You Are

The loss is a single number that captures the network's mistake. When the prediction matches the target, the loss is zero. When they diverge, the loss grows. Training means making this number as small as possible.

But the loss only tells you the total error. It does not tell you which weights caused it. That is the job of the backward pass.

The Backward Pass: Who Is Responsible?

After the forward pass, error flows backward. At each connection, the algorithm asks: if this weight were slightly different, how much would the loss change? That sensitivity is the gradient.

The chain rule makes this tractable. At each node, the incoming gradient gets multiplied by the local gradient, which is the derivative of whatever that node computed. For a sigmoid node, the local gradient is the slope of the sigmoid curve at the current operating point. Gradients compose by multiplication as they flow backward through the network.

Figure 3. The backward pass. Red pulses carry gradients from the output back to the inputs. At each node, the incoming gradient is multiplied by the local derivative (sigmoid slope). Edge labels show the computed gradient for each weight.

The chain rule is the engine of backpropagation. Each node only needs to know its own local derivative. Multiply that by the gradient arriving from downstream, and you have the gradient to pass upstream. No node needs a global view of the network.

Weight Updates: Adjusting Based on Responsibility

Once every weight has a gradient, the update rule is simple: shift each weight in the direction that reduces the loss, proportional to how much that weight contributed to the error. A large gradient means the weight has a big effect on the loss, so it gets a bigger adjustment. A small gradient means the weight barely matters right now, so it barely moves.

The learning rate controls the step size. Too large and the weights overshoot. Too small and learning takes forever.

Credit Assignment

Backpropagation is credit assignment. After seeing the outcome, the algorithm traces back through every connection and asks: did you help or hurt? Connections that contributed to the error get weakened. Connections that pushed toward the right answer get strengthened.

This is not trial and error. The gradients give precise direction. Each weight knows exactly which way to move and roughly how far. That precision is what makes neural networks trainable at all. Without it, adjusting thousands of weights would be like searching in the dark.

The math is just the chain rule from calculus, applied systematically. The insight is organizational: by structuring computation as a graph of differentiable operations, you get the gradients for free. Any function you can compute forward, you can differentiate backward. That is the idea that makes deep learning possible.
