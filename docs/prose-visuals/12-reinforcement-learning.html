Reinforcement Learning

Learning through trial, error, and reward

How do you learn to navigate a city you've never visited? Not from a textbook. You try routes, notice which ones work, hit dead ends, backtrack. Over dozens of trips you build a mental map of good decisions: turn left at the bakery, avoid the construction on 5th. Nobody told you the optimal route. You discovered it through experience.

Reinforcement learning works the same way. An agent sits in an environment, takes actions, and receives rewards or penalties. It has no teacher. It has no labeled examples. It just has consequences.

The Setup: States, Actions, Rewards

Consider a grid world. An agent starts in one cell and must reach a goal. Some cells are pits that deal heavy penalties. Every step costs a small amount, a tax on indecision. The agent can move up, down, left, or right. That is the entire problem.

Even in this tiny world, finding the optimal path requires thought. You need to balance directness (fewer steps, less cost) against safety (avoiding pits). Now imagine the grid is a thousand cells wide, or the rewards are hidden, or the actions are uncertain. That is where learning algorithms earn their keep.

Q-Learning: Updating Beliefs from Experience

Q-learning assigns a value to each (state, action) pair. The Q-value for "being in cell (3,2) and moving right" represents the agent's current belief about how good that action is from that position. Initially these values are all zero: the agent knows nothing.

Then it explores. Each episode, the agent starts at the beginning, picks actions, collects rewards, and updates its Q-values. The update rule is simple: after taking action a in state s, receiving reward r, and landing in state s', adjust:

Q(s, a) = Q(s, a) + α · [r + γ · max Q(s', a') - Q(s, a)]

The agent nudges its belief toward reality. The learning rate α controls how fast. The discount factor γ controls how much future rewards matter. Episode after episode, the Q-values converge. Cells near the goal develop high values. Cells near pits develop low ones. And the agent learns without anyone telling it the answer.

Exploration vs. Exploitation

The agent faces a dilemma every step. Should it take the action it currently believes is best (exploit), or try something new to potentially discover a better path (explore)? Too much exploitation and the agent locks onto the first decent route, missing shorter ones. Too much exploration and it wanders aimlessly, never capitalizing on what it has learned.

The parameter ε (epsilon) controls this tradeoff. With probability ε the agent picks a random action; otherwise it picks its current best. Watch two agents learn the same grid side by side: one cautious exploiter, one bold explorer.

The Policy: What the Agent Learned

After learning, the Q-values encode a complete strategy: for every cell, the agent knows which direction to go. This strategy is the policy. We can visualize it two ways.

The arrow field shows the best action in each cell. Arrow opacity reflects confidence: a bold arrow means one action is clearly superior, a faint arrow means the agent is nearly indifferent. The value heatmap shows how good each cell is (the maximum Q-value). Bright cells are close to reward; dark cells are far or dangerous. Both views reveal the same learned knowledge from different angles.

The value heatmap reveals the geometry of the problem. Value radiates outward from the goal along safe corridors and drops sharply near pits. The agent has learned a topographic map of reward, and the policy is simply "walk uphill."

From Grid World to the Real World

This grid is a toy, but the principles are real. The same algorithm that learns to navigate seven cells can learn to play Atari games, control robotic arms, manage power grids, and optimize data center cooling. The state space grows, the action space grows, and the Q-table becomes a neural network. But the core loop stays the same: try, observe, update, repeat.

Reinforcement learning is how you build intuition from experience. Not by being told the answer, but by discovering which actions lead to rewards. Every time you choose a restaurant, adjust your commute, or decide how to phrase a difficult email, you are running a version of this algorithm. The world is the grid. Your choices are the actions. And the rewards, for better or worse, are whatever happens next.
