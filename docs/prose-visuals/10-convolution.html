Convolution

How sliding kernels detect features in images and signals.

A kernel is a question you ask every patch of the input: do you look like this?

Place a small template over a region of the data. Multiply point by point, sum the results. Slide over one position. Repeat. That sum, the dot product between the kernel and the local patch, is the answer to the question at that location. A large value means strong resemblance. A value near zero means no match.

This is convolution. It is the mechanism at the heart of convolutional neural networks, and it works because it encodes a simple assumption: the same pattern can appear anywhere in the input. A horizontal edge in the top-left corner looks the same as a horizontal edge in the bottom-right. One small kernel can find all of them.

The Sliding Dot Product

Start with one dimension. A signal is just a list of numbers, a sequence of samples from some measurement. A kernel is a shorter list. Convolution slides the kernel across the signal, computing a dot product at every position, producing a new signal: the output.

Below, a wavy signal with 50 samples. A small kernel slides across it. At each position, the kernel values multiply the corresponding signal values, and those products sum to one output value. The kernel never changes as it slides. Convolution applies the same operation everywhere. This is called translation equivariance: if the input shifts, the output shifts by the same amount. The pattern detector does not care where the pattern is.

The Same Idea on a Grid

Images are two-dimensional. Each pixel has a brightness value arranged on a grid. A 2D convolution works the same way as 1D, but the kernel is now a small square, typically 3x3. It slides across every row and column, computing a weighted sum at each position to produce a feature map.

Edges, Textures, Parts

One layer of convolution detects simple features: edges, gradients, spots of brightness. But a convolutional neural network stacks many layers. The output feature maps of one layer become the input to the next. This composition is what gives CNNs their power.

Layer 1 detects edges. Layer 2 combines edges into textures and patterns. Layer 3 combines textures into recognizable parts. Each layer builds on the abstractions of the layer before it, forming a hierarchy of increasingly complex features.

Click any feature map in Layer 3 below. It highlights the Layer 2 features that feed into it, which in turn highlights the Layer 1 features that feed into those. You can trace the hierarchy of abstraction backward, from parts to textures to edges.

Figure 3. Feature hierarchy in a CNN. Click a feature in a later layer to trace its inputs backward through the network. Edges combine into textures, textures into parts.

The network learns which kernels to use. During training, gradient descent adjusts the kernel values so that early layers learn edge detectors, middle layers learn texture detectors, and deep layers learn part detectors. Nobody programs these features by hand.

Padding, Stride, and Pooling

Three hyperparameters control the geometry of convolution: how the output size relates to the input size.

Padding adds zeros around the border of the input. Without padding, the output shrinks (a 3x3 kernel on an 8x8 grid produces 6x6). With padding of 1, the output stays the same size as the input.

Stride controls how far the kernel jumps between positions. Stride 1 moves one pixel at a time. Stride 2 skips every other position, cutting the output size in half.

Pooling is a separate operation applied after convolution. It divides the feature map into small blocks (typically 2x2) and reduces each block to a single value, either the maximum or the average. This further reduces spatial size.

Stride and pooling both reduce spatial size, but they do it differently. Stride skips positions during convolution, so the kernel sees fewer patches and computation is cheaper. Pooling runs after a full convolution pass, combining neighboring values. Max pooling preserves the strongest activation in each neighborhood. Average pooling blurs. The choice matters for what information survives.

Why It Works

Convolution encodes one strong assumption: the same pattern can appear anywhere in the input. A vertical edge at pixel (3, 4) is the same feature as a vertical edge at pixel (50, 80). This weight sharing means CNNs need far fewer parameters than fully connected networks, and they generalize better on spatially structured data like images, audio spectrograms, and time series.

The hierarchy of layers turns local features into global understanding. Edges become textures, textures become parts, parts become objects. Each layer expands the receptive field, the region of the original input that influences a single output value, without losing the pattern-matching discipline of the kernel.

Convolution is not the only way to process spatial data. Attention mechanisms (as in vision transformers) drop the locality assumption and let every position attend to every other. But convolution remains efficient, well-understood, and remarkably effective. Most modern architectures still include convolutional layers somewhere in the pipeline.
