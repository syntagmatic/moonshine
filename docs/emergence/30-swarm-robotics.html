Swarm Robotics

Simple robots with no central controller that collectively sort, assemble, and explore.

No robot knows the plan. That is the first thing to understand about swarm robotics. Each machine in the group has a short sensor range, a small set of reactive rules, and no memory of the global state. If it senses a neighbor nearby, it might turn toward it. If it bumps into an obstacle, it backs up and pivots. That is the entire program.

And yet the group does things no individual could. Forty robots that each follow "move toward visible neighbors, avoid collisions" will aggregate into a tight cluster. Robots that pick up isolated objects and drop them among similar objects will sort a mixed field into neat piles. Robots that compare their neighbor count to a target will flow into circles, lines, and hexagons. The collective behavior is not coded anywhere. It emerges from the interactions.

1. Aggregation

The simplest swarm behavior: come together. Each robot scans for neighbors within its sensor range. If it sees any, it steers toward their average position while keeping a minimum separation distance to avoid collisions. If it sees nobody, it wanders randomly. Start the simulation below and watch 40 scattered robots condense into a cluster. Then drag the sensor range slider and notice how the dynamics change. A large range means fast, direct aggregation. A small range means slow, organic clumping through local chains.

2. Collective Sorting

Scatter two kinds of tokens across a field, red and blue, in random positions. Let robots wander through this mess. A robot that walks over a token checks the local neighborhood: how many tokens of the same color are nearby? If the token is isolated (few same-colored neighbors), the robot picks it up and carries it. If the robot is carrying a token and walks into an area dense with the same color, it drops the token there. That is the entire algorithm. Over time, mixed piles self-organize into sorted clusters. No robot plans the sorting. It just happens.

3. Shape Formation

This is where things get impressive. Robots start in random positions and must arrange themselves into a target shape using only local information. Each robot knows the desired shape as a distance function: for any position, the function returns how far that position is from the nearest point on the target boundary. A robot measures its distance to visible neighbors, computes a local gradient of the shape function, and nudges itself toward the boundary. 4. Resilience

The real test of a swarm is damage. Click on the canvas below to destroy robots near your cursor. The remaining swarm will close the gap and reform the shape. A centralized system with a master controller would need to detect the failure, recompute assignments, and redistribute commands. The swarm just keeps running its local rules. The Kilobot project at Harvard demonstrated 1,000 simple robots self-assembling into arbitrary shapes using only local communication via infrared. Each robot costs about $14 and has less processing power than a calculator.

Every simulation on this page runs the same basic loop: sense local neighbors, apply a simple rule, move. No robot has a map. No robot knows how many others exist. No robot receives instructions from a central planner. The collective behavior is not written in any single robot's code. It lives in the interactions. The swarm is more than the sum of its parts, not because the parts are clever, but because the right local rules, applied in parallel by many agents, generate order from noise.
