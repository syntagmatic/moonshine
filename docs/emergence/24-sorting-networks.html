Sorting Networks

A fixed wiring of compare-and-swap operations sorts any input, and the optimal network for a given size is still an open problem.

A sorting network is a fixed sequence of pairwise comparisons. Each comparison takes two elements and puts the smaller one on top. When the comparisons are wired correctly, any permutation comes out sorted. The network does not adapt to the input. The same wires handle every case. Order emerges from structure, not intelligence.

Unlike the sorting algorithms you learn in a CS class (quicksort, mergesort), a sorting network has no branches and no conditionals. It cannot look at the data and decide what to do next. Every comparison is hardwired in advance. This makes sorting networks natural candidates for hardware implementation and parallel processing, because many comparisons can happen at the same time.

1. Compare and Swap

The fundamental operation is the comparator. It connects two wires. Values arrive on both wires. If they are out of order, the comparator swaps them. If they are already in order, nothing happens. Click "Randomize" below to put new values on the wires and watch the comparator do its work.

Figure 1. A single comparator connects two wires. When the top value is larger than the bottom, it swaps them. The smaller value always ends up on top.

One comparator can only handle two elements. But chain several together and they can sort more. Below is a network of three comparators that sorts any permutation of three elements. The first two comparators work on different pairs, and the final one cleans up whatever is left. Click "Randomize" to try different inputs.

Figure 2. A 3-element sorting network. Three comparators are enough to sort any arrangement of three values. Values flow from left to right, swapping at each comparator when out of order.

Now try designing a sorting network yourself. Click between two wires to place a comparator. The network will be tested against all permutations (for small n) or random samples (for larger n). Finding the optimal sorting network (fewest comparators) for a given number of inputs is an open problem for n > 12. The answer for n = 25 is unknown. This is one of the simplest questions in computer science that remains unsolved. The known optima for small sizes: n=1 needs 0, n=2 needs 1, n=3 needs 3, n=4 needs 5, n=5 needs 9, n=6 needs 12. Each was proven by exhaustive search or careful argument. Beyond n=12, we only have upper bounds.
