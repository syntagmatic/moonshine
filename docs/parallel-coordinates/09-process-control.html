Process Control

Industrial sensor data across eight variables. Parallel coordinates let operators see the full state of a chemical reactor in one view, detect drift, and trace fault propagation through the system.

A chemical reactor is instrumented with sensors everywhere. Inlet temperature, outlet temperature, pressure, flow rates, chemical concentrations, catalyst level, pH. Each sensor produces a time series. A control room operator needs to monitor all of these simultaneously, detect when something drifts out of specification, and figure out which variable is the root cause.

This is exactly the situation parallel coordinates were designed for. Each time step is a single observation across all variables. That observation maps to a polyline. The full history of normal operation produces a bundle of polylines. Deviations from normal appear as polylines that escape the bundle — the same outlier signature from earlier, but now in a real-time monitoring context where the pattern of deviation points toward the fault.

The Chemical Process Monitor

The display below shows a parallel coordinates view of a chemical reactor with eight measured variables. The faded bundle represents 200 historical readings during normal operation. The bold polyline is the current state of the reactor, and you can drag it on any axis to simulate changing conditions.

Each axis also shows upper and lower specification limits as dashed lines. When the current reading falls outside normal operating range on any axis, that axis label turns red. This is an operator's dashboard: one glance reveals whether the process is healthy or stressed.

Drag the blue dot on any axis to change the current reading. Watch axis labels turn red when the value moves outside the normal bundle.

Figure 1. Chemical process monitor. The faded bundle is 200 normal readings. The bold polyline is the current state, draggable on each axis. Dashed lines mark specification limits. Axis labels turn red when the current reading exceeds spec.

Key principle. Normal operation is a cluster in 8-dimensional space. The parallel coordinates bundle makes that cluster visible. Faults are outliers whose polylines deviate from the bundle on specific axes — and the pattern of deviation encodes the nature of the fault.

Drift Detection

Not all faults happen suddenly. Some processes drift over time: a catalyst slowly deactivates, a heat exchanger fouls, a sensor calibration slips. In a table of numbers, a slow drift in one or two variables out of eight is easy to miss. In parallel coordinates, it becomes visible if you color the readings by time.

Below, 50 time-ordered readings are colored from blue (earliest) to red (most recent). A simulated drift causes outlet_temp and concentration_B to shift gradually over time. The time coloring makes the drift visible as a gradient across the bundle on the affected axes, while unaffected axes show blue and red interleaved.

Figure 2. Time-ordered readings colored blue (early) to red (late). Outlet temperature and concentration B drift over time. Toggle to compare with a stable process.

Cause-and-Effect Ordering

Axis ordering is decisive here, just as it was in explainer 04. In a chemical reactor, variables are not independent. Inlet temperature drives outlet temperature. Temperature affects reaction kinetics, which determines concentrations. Placing causally related variables on adjacent axes makes these chains visible as smooth, parallel bundles between the axes. Placing unrelated variables adjacent produces a tangled mess.

The figure below starts with an arbitrary axis order. Click "Suggest order" to rearrange axes using a greedy correlation-based algorithm: start with the variable pair that has the highest absolute correlation, then repeatedly add the variable most correlated with the current endpoint. The result places causally related sensors adjacent, making the process chain legible.

Drag axis labels to reorder manually, or click "Suggest order" to let the correlation algorithm arrange them.

Figure 3. Axis reordering by pairwise correlation. The algorithm places strongly correlated variables adjacent, making the causal chain inlet_temp → outlet_temp → concentration_A → concentration_B visible as smooth bundles.

Causal chains become visible. When axes are ordered by correlation, transitions between adjacent axes are smooth. In a drift event, you can read the chain: which variable shifted first, which followed, and which was unaffected. This is how operators trace a fault from symptom back to root cause.

Alarm Regions and Brushing

An alarm fires when a variable exceeds its limit. But a single alarm tells you nothing about the system state across other variables. Brushing in parallel coordinates lets you select the alarm readings on one axis and immediately see which other axes are also abnormal for those readings. This is the multi-dimensional view that a threshold alarm on a single variable cannot provide.

Below, 200 normal readings are gray. Ten alarm readings, generated to violate limits on one or more axes, are red. Brush (click and drag) on any axis to select a range. Only polylines passing through the brushed range are highlighted. Brush the alarm region on one axis to see the correlated abnormalities on others.

Click and drag vertically on any axis to brush. Polylines passing through the brushed range are highlighted.

Figure 4. Normal readings (gray) and alarm readings (red). Brush any axis to select a range. Try brushing the extreme region on outlet_temp to see which other variables are simultaneously abnormal.

Process control combines both fundamental parallel coordinates patterns: normal operation forms a cluster, and faults produce outlier signatures with specific dimensional fingerprints. The axis-ordering and brushing tools from the geometric foundations become, in this applied context, the operator's instruments for reading the state of a complex system and diagnosing what went wrong.
