Axis Order Is Everything

Permuting the axes changes what you can see. The same data tells different stories depending on which dimensions sit next to each other.

Parallel coordinates plot N variables on N vertical axes. Each observation is a polyline connecting its values across the axes. But unlike a scatterplot matrix, where every pair of variables gets its own panel, a parallel coordinates plot only reveals relationships between adjacent axes. The crossing patterns that encode correlation (as we saw in explainer 01) are only visible between neighbors.

This means the order of the axes is not cosmetic. It is the primary design decision. Change the axis order, and you change which relationships are visible and which are hidden.

Only Adjacent Pairs Are Visible

A scatterplot matrix shows all pairwise relationships at once. A parallel coordinates plot with the same N variables shows only N−1 adjacent pairs. This is the fundamental trade-off: parallel coordinates can handle many more dimensions, but at the cost of making most pairwise relationships invisible in any single ordering.

The figure below makes this concrete. The correlation matrix on top shows all pairwise absolute correlations. The blue cells are the ones currently visible as adjacent pairs in the parallel coordinates plot below. Reorder the axes and watch which cells light up.

The symmetry trade-off. Parallel coordinates sacrifice the symmetry of the scatterplot matrix (where all pairs are visible) for the ability to display many dimensions at once. Axis ordering is the primary design decision that determines what the viewer will see.

Greedy Correlation-Based Ordering

If correlations between adjacent axes are what we see, a natural strategy is to place the most strongly correlated pairs next to each other. The greedy nearest-neighbor algorithm works as follows:

1. Compute the absolute correlation between every pair of variables.
2. Start with the pair that has the highest |correlation|.
3. At each end of the current chain, find the unplaced variable most correlated with the endpoint. Add it to whichever end gives the higher correlation.
4. Repeat until all variables are placed.

Press the button below to watch the algorithm build the ordering step by step.

Figure 4. Greedy correlation-based axis ordering. The algorithm starts with the most correlated pair and greedily extends the chain. At each step, the newly placed axis animates into position. The result clusters the strongest correlations into adjacent positions.

The greedy algorithm does not guarantee the globally optimal ordering (that would require checking all permutations), but it typically finds a good one. It is the same heuristic used for the travelling salesman problem on the correlation graph, treating |correlation| as a distance to maximize.

The key insight. There is no single "correct" axis order. Different orderings reveal different structures. Correlation-based ordering is a sensible default because it maximizes the amount of structure visible as crossing patterns between adjacent axes. But manual exploration often finds orderings that highlight features the algorithm misses, such as cluster boundaries or outlier signatures that depend on specific variable pairs.
